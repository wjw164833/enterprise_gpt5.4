# enterprise_gpt5.4

一个已封存的多端邀请函系统项目，包含后端、管理前端和微信小程序三个部分。

> 当前仓库用于归档备份，不再作为持续开发中的正式项目维护。

---

## 项目简介

这是一个“企业级邀请函系统”原型项目，目标是提供一套多端邀请函解决方案，包括：

- 邀请函创建与管理
- 宾客回复与留言互动
- 席位管理
- 数据统计分析
- 支付/订阅能力
- 微信端访问与互动

项目后期存在较多环境依赖、未完成功能和历史遗留问题，已不适合继续直接迭代，因此进行整体封存。

封存原因
项目后期出现了以下情况：
多端工程分散，维护成本高
环境依赖较多，启动链路复杂
存在较多历史遗留问题
一部分功能未完成，一部分功能虽可运行但不稳定
继续推进的修复与重构成本已经高于直接重建的收益
因此决定将该项目整体归档保存，作为历史备份与结构参考。
